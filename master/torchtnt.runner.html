class torchtnt.runner.AutoUnit(*, module: Module, optimizer: Optimizer, lr_scheduler: Optional[_LRScheduler] = None, step_lr_interval: Literal['step', 'epoch'] = 'epoch', device: Optional[device] = None, log_frequency_steps: int = 1000, precision: Optional[Union[str, dtype]] = None, gradient_accumulation_steps: int = 1, detect_anomaly: bool = False, clip_grad_norm: Optional[float] = None, clip_grad_value: Optional[float] = None, swa_params: Optional[SWAParams] = None)¶

The AutoUnit is a convenience for users who are training with stochastic gradient descent and would like to have model optimization handled for them. The AutoUnit subclasses TrainUnit, EvalUnit, and PredictUnit, and implements the train_step, eval_step, and predict_step for the user.

For the train_step it runs: * forward pass as part of the loss computation * backward pass * optimizer step

For the eval_step it only runs forward and loss computation.

For the predict_step it only runs forward.

To benefit from the AutoUnit, the user must subclass it and implement the compute_loss method, and optionally the update_metrics and log_metrics methods. Then use with the train(), evaluate(), predict(), or fit() entry point as normal.

For more advanced customization, directly use the TrainUnit, EvalUnit, and PredictUnit interfaces. :param module: module to be used during training. :param optimizer: optimizer to be used during training. :param lr_scheduler: lr_scheduler to be used during training. :param step_lr_interval: whether to step lr_scheduler every step or every epoch. Defaults to every epoch. :param device: the device to be used. :param log_frequency_steps: how often to log in terms of steps (parameter updates) during training. :param precision: the precision to use in training, as either a string or a torch.dtype. :param gradient_accumulation_steps: how many batches to accumulate gradients over. :param detect_anomaly: whether to enable anomaly detection for the autograd engine https://pytorch.org/docs/stable/autograd.html#anomaly-detection :param clip_grad_norm: max norm of the gradients for clipping https://pytorch.org/docs/stable/generated/torch.nn.utils.clip_grad_norm_.html :param clip_grad_value: max value of the gradients for clipping https://pytorch.org/docs/stable/generated/torch.nn.utils.clip_grad_value_.html :param swa_params: params for stochastic weight averaging https://pytorch.org/docs/stable/optim.html#stochastic-weight-averaging

abstract compute_loss(state: State, data: TData) → Tuple[Tensor, Any]¶

The user should implement this method with their loss computation. This will be called every train_step/eval_step.

Parameters:

  • state – a State object which is passed from the train_step/eval_step

  • data – a batch of data which is passed from the train_step/eval_step

Returns:

Tuple containing the loss and the output of the model

eval_step(state: State, data: TData) → Tuple[Tensor, Any]¶

Optionally can be decorated with @torch.inference_mode() for improved peformance.

log_metrics(state: State, step: int, interval: Literal['step', 'epoch']) → None¶

The user should implement this method with their code to log metrics. This will be called: * every train_step based on log_frequency_steps and how many parameter updates have been run on the model * in on_train_epoch_end and on_eval_epoch_end

Parameters:

  • state – a State object which is passed from train_step/on_train_epoch_end/on_eval_epoch_end

  • step – how many steps have been completed (i.e. how many parameter updates have been run on the model)

  • interval – whether log_metrics is called at the end of a step or at the end of an epoch

move_data_to_device(state: State, data: TData) → TData¶

The user can override this method with custom code to copy data to device. This will be called at the start of every train_step/eval_step/predict_step. By default this uses the utility function copy_data_to_device().

Parameters:

  • state – a State object which is passed from the train_step/eval_step/predict_step

  • data – a batch of data which is passed from the train_step/eval_step/predict_step

Returns:

A batch of data which is on the device

torchtnt.runner.fit(state: State, unit: TrainUnit[TTrainData], *, callbacks: Optional[List[Callback]] = None) → None¶

The fit entry point interleaves training and evaluation loops.

Parameters:

  • state – a State object containing metadata about the fitting run. init_fit_state() can be used to initialize a state object.

  • unit – an instance that subclasses both TrainUnit and EvalUnit, implementing train_step() and eval_step().

  • callbacks – an optional list of callbacks.

torchtnt.runner.init_fit_state(train_dataloader: Iterable[TTrainData], eval_dataloader: Iterable[TEvalData], max_epochs: Optional[int] = None, max_steps: Optional[int] = None, max_train_steps_per_epoch: Optional[int] = None, max_eval_steps_per_epoch: Optional[int] = None, evaluate_every_n_steps: Optional[int] = None, evaluate_every_n_epochs: Optional[int] = 1) → State¶

Helper function that initializes a State object for fitting.

Parameters:

  • train_dataloader – dataloader to be used during training.

  • eval_dataloader – dataloader to be used during evaluation.

  • max_epochs – the max number of epochs to run for training. None means no limit (infinite training) unless stopped by max_steps.

  • max_steps – the max number of steps to run for training. None means no limit (infinite training) unless stopped by max_epochs.

  • max_train_steps_per_epoch – the max number of steps to run per epoch for training. None means train until the dataloader is exhausted.

  • evaluate_every_n_steps – how often to run the evaluation loop in terms of training steps.

  • evaluate_every_n_epochs – how often to run the evaluation loop in terms of training epochs.

Returns:

An initialized state object containing metadata.
